Epurer un modèle

Pendant le processus créatif et itératif de modélisation, des éléments du modèle pourraient être créés s'ils ne sont plus nécessaires une fois le modèle achevé. Open ModelSphere offre une fonction d'épuration pour trouver et effacer ces éléments.

En utilisant l'option d'épuration, une nouvelle fenêtre apparaîtra pour vous énumérer les occurences du modèle qui ne répondent pas aux critères de cohérence et aussi les liens vers ceux-ci dans la fenêtre Explorateur et les diverses fenêtres de propriétés.

La fonction n'épuration est disponible pour les modèles de processus, de données, d'items communs ainsi que pour les bibliothèques d'opérations.

Pour utiliser la fonction Epurer le modèle :

  1. Sélectionner le modèle ou un diagramme de ce modèle dans l'explorateur. (il est à noter que l'opération est effectuée sur le modèle et non sur un seul diagramme sélectionné)
  2. Du menu Outils, sélectionnez Épurer le modèle . La fenêtre suivante apparaîtra :

L'épuration peut être faite de trois façons :

  • sur une occurence en particulier via l'icône .
  • sur l'ensemble des occurences lié à une règle particulière.
  • sur toutes les occurences énumérées dans le rapport.
Vous pouvez faire également effectuer les changements à partir de la fenêtre de propriétés de chaque objet; utiliser le lien .
 
 

Règles utilisées dans l'épuration

Modèle de données et de domaines

  1. Une table doit avoir au moins un colonne (avertissement).
  2. Une table doit être liée à au moins une autre table (avertissement).
  3. Une clé primaire/unique doit s'appliquer à une colonne ou une dépendence.
  4. Une clé étrangère doit s'appliquer à une colonne.
  5. Un index doit avoir au moins un élément indexé (une colonne).
  6. Un déclencheur doit avoir un corps d'instructions être défini sur une colonne
  7. Une contrainte doit s'appliquer à une colonne
  8. Un domaine doit être lié à un type de donnée.
  9. Un domaine doit être utilisé par au moins un objet.


Bibliothèque d'opérations

  1. Une procédure doit contenir des instructions (avertissement).
  2. Un paramètre doit avoir un type.


Modèle de processus

  1. Un processus doit avoir une représentation graphique (avertissement).
  2. Une entité externe doit avoir une représentation graphique (avertissement).
  3. Un dépôt doit avoir une représentation graphique (avertissement).

Modèle d'items communs
  1. Un item commun doit être lié à au moins une colonne (avertissement).
  2. Un item commundoit avoir un type.
