Créer un modèle de domaines


Un domaine constitue l'ensemble des valeurs qu'un attribut, une colonne ou un item commun peut prendre. Vous devez donc lier un domaine à chaque attibut, colonne et item commun,que ce soit lors de la création de l’objet ou par la suite.

Open ModelSphere offre aussi des modèles de domaines. Il y a trois façons de créer ce type de modèle :
  1. Du menu principal, choisissez Édition > Ajouter > Modèle de domaines > Système cible de votre choix.

  2. Dans la fenêtre d’explorateur, cliquez avec le bouton droit sur Projet > Ajouter > Modèle de domaines > Système cible de votre choix.

  3. De la barre d’outils Édition, cliquez sur l’outil Ajouter > Modèle de domaines > Système cible de votre choix.

Selon les configurations par défaut, le diagramme du modèle de domaines n’est affiché que sur demande. Utilisez l'outil pointeur, choisissez le modèle de domaines dans la fenêtre d’explorateur et choisissez Ajouter > Diagramme. Si vous désirez modifier les configurations par défaut, référez-vous à la section Modifications des options.

Pour ajouter des objets au modèle de domaines : Sélectionnez le modèle de domaines dans la fenêtre explorateur, et choisissez le menu principal Edition > Ajouter > Domaine.



Nous vous recommandons de nommer vos domaines aussitôt que vous les avez créés. Ainsi, vous éviterez toute confusion lorsque le modèle se complexifiera. Pour ce faire, utilisez la fonction d’édition graphique du contenu du domaine.

Si la propriété Propagé est cochée les valeurs des descripteurs
  • Longueur
  • Nombre de décimales
  • Valeur nulle possible
  • Type source
seront utilisées de manière uniforme. Ils seront propagés aux descripteurs d'attributs correspondant aux colonnes, item communs et domaines auxquelles ils sont liés.
