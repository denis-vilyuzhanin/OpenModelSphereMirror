Redémarrage en mode sûr


Open ModelSphere a été conçu pour être une application très stable. Il est assez rare que l'application toute entière plante sans aucune possibilité de recouvrement. Si cela advient, c'est la plupart du temps pour des raisons externe à ModelSphere. Néanmoins, si une configuration particulière rends ModelSphere instable, il y a la possibilité de désactiver les fonctionalités qui peuvent rendre l'application instable et de redémarrer en mode sûr.

Pour invoquer cette fonctionnalité, choisissez Aide > Avancé > Redémarrage en mode sûr dans le menu principal. Vous aurez l'opportunité d'activer ou de désactiver cinq options qui peuvent causer de l'instabilité :
  • Désactiver DirectDraw: DirectDraw, une composante de Sun utilisé par les librairies graphique Swing, peut faire planter la machine virtuelle Java. Cela peut advenir avec certaines cartes vidéo (ATI en particulier). DirectDraw peut être désactivé si l'application éprouve ce genre de problèmes.

  • Effacer les préférences: Parce que ModelSphere conserve ses préférences dans un fichier texte (.properties), elles peuvent être accidentellement éditées et des préférences se voir assigner des valeurs non-valide. Ceci peut rendre ModelSphere disfonctionnel. Les préférences altérées, peuvent être effacées, ce qui entrainera la remise aux valeurs par défaut lors du prochain démarrage.

  • Utiliser Metal comme Apparence (Look and Feel): Swing supporte plusieurs apparences (Look and Feel). Si une apparence fait planter ModelSphere, il est possible d'utiliser l'apparence Metal au prochain démarrage. (Metal étant l'apparence la plus stable).

  • Ne charger aucun module d'extension: ModelSphere est un produit ouvert, et il est possible à tous d'ajouter des modules d'extension personnalisés. Si un module d'extension est disfonctionnel, cela peut faire planter ModelSphere. Il est toujours possible dans ce cas de forcer ModelSphere à redémarrer sans charger aucun module d'extension.

  • Passer à la locale Anglaise: Cette option permet de redémarrer l'application en langue anglaise. L'anglais est en principe plus sûr car c'est dans cette langue que le développement de ModelSphere se fait préalablement.
